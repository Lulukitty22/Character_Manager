Conversions

What this is: conversion rules let resources transmute into each other in play. Each rule defines a source → destination cost. How it works: editor-side you save rules here; viewer-side, players see clickable Convert buttons next to the affected resource. Click Save rule to commit a row; Re-save to update an existing one. Used for: Sorcerer Flexible Casting (SP ↔ slots), Carol's Memory Burn (Spell Memories → any slot), or any custom alchemical rule.

Editor mockup — round 3:
• Unified row layout — every list row (class, variant, relationship, feat, spell, item, resource) uses the same template: title + source-badge + chip-row + ⚙/× icons + collapsible edit drawer. Library-linked vs inline differs only in the source badge and the edit-drawer contents (override fields vs full edit form). New rows added via + button or library picker use the same template.
• Saving throw pips wire to the displayed modifier and the live preview.
• Hover ability blocks + save mods for tooltip breakdowns of where each stat is used.

• AC mode — adding a new mode now appends a chip in the live preview's AC Modes row.

• Conversion rule rows include cost/yield chips, and the help-tip above explains what saving does, what "Re-save" means, and what shows up in the viewer.
• Resource rows show progress bars in their chip row (current/max).
• Two add buttons per card (where applicable): + from library and + inline — distinct actions, kept separate on purpose.
• Open in a real browser to actually click around — the launch preview is static.

How to get a PAT

  1. Visit github.com/settings/personal-access-tokens
  2. Click "Generate new token (fine-grained)"
  3. Repository access: Only select repositories → Character_Manager
  4. Permissions: Contents: Read and write
  5. Copy token, paste above, click Test Connection
